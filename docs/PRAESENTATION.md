# Slide 2 – Das Problem

## Heute in deutschen Werkstätten

- 📓 **Papier-Terminbuch** – Termine verloren, doppelt gebucht, unlesbar
- 📞 **Telefon als einziger Buchungskanal** – Anrufe gehen verloren, keine Abende/Wochenenden
- 💻 **5–10 Tools parallel** – Excel, Word, Outlook, Teamviewer, DATEV, Mail …
- ⏱️ **Zeitverlust** – 30 % der Mechaniker-Zeit ist nicht an Fahrzeugen, sondern in Verwaltung
- ❌ **Keine Auswertungen** – "Wie läuft es? Keine Ahnung, gefühlt gut."
- 🤷 **Kunden-Erwartungen steigen** – sie wollen online buchen, Erinnerungen bekommen, Transparenz

# Slide 13 – Intelligente Planung

## Puffer vor, Arbeit, Puffer nach

```
│← Puffer vor  ←│←    Arbeitszeit    →│← Puffer nach →│
   Fahrzeug-         Eigentliche         Probefahrt,
   Annahme            Reparatur          Übergabe
   (15 Min)            (120 Min)          (20 Min)
```

**Beispiel: Bremsen VA komplett (BRE-03)**
- Puffer vor: 15 Min (Fahrzeug-Annahme, Laufzettel)
- Arbeitszeit: 90–120 Min
- Puffer nach: 20 Min (Probefahrt, Waschen, Übergabe)
- **Gesamt-Slot: 155 Min**

→ Realistische Planung, weniger Verzögerungen, weniger unzufriedene Kunden.

# Slide 18c – Digitales Arbeitsprotokoll

## Kennzeichen-KI · Checkliste · Unterschrift

*[Screenshot: Tablet zeigt Foto-Kennzeichen + KI-Erkennung, daneben Checkliste]*

**Der neue Arbeitsschein – digital, rechtssicher, lückenlos.**

- **Schritt 1 – Start per Kennzeichen-Foto**
  - Mitarbeiter fotografiert Kennzeichen mit Tablet/Handy
  - KI liest Kennzeichen automatisch aus
  - Abgleich mit Termin: ✓ richtiges Fahrzeug
  - Kilometerstand wird erfasst

- **Schritt 2 – Checkliste während/nach der Reparatur**
  - Automatisch passende Prüfpunkte je Leistung
  - 10 Standard-Vorlagen: Bremsen · Öl · Reifen · Elektrik · Klima · HU · Fahrwerk · Karosserie · Inspektion · Übergabe
  - Status: ✓ OK · ✗ Nicht OK (mit Pflicht-Notiz) · n/a
  - Messwerte in Volt, mm, Bar frei eintragbar

- **Schritt 3 – Digitale Unterschrift**
  - Mitarbeiter unterschreibt per Finger auf dem Tablet
  - Auftrag automatisch abgeschlossen mit echter Zeit
  - Protokoll druckbar / als PDF an Kunden

**Vorteile:**

| Früher (Papier) | Jetzt (digital) |
|-----------------|-----------------|
| Arbeitsschein vergessen → kein Nachweis | Lückenlos, automatisch |
| Vergessene Prüfpunkte | Pflicht-Checks, keine Chance zu vergessen |
| Unleserliche Handschrift | Sauberer Druck mit Fotos + Signatur |
| Kunde bezweifelt die Arbeit | Beweis: Fotos, KM, Unterschrift, Zeitstempel |
| Qualitätsprüfung nur stichprobenhaft | Jeder Auftrag dokumentiert |

→ **Versicherung + Gewährleistung + Kundenvertrauen** – alles mit einem Tool.

# Slide 18d – Digitales Übergabeprotokoll

## Der Kunde unterschreibt, dass alles stimmt

*[Screenshot: Tablet mit Checkliste „Schlüssel ✓, Papiere ✓, Zubehör ✓" + Kunden-Unterschrift]*

**Schluss mit „das war ja gar nicht so!" – jede Auslieferung rechtssicher.**

- **Schritt 1 – Übergabe-Inhalte**
  - 🔑 Anzahl Schlüssel zurück (Soll vs. Ist)
  - 📄 Fahrzeugpapiere (Fahrzeugschein, TÜV …)
  - 🎒 Zubehör + persönliche Gegenstände
  - 🛣 End-Kilometerstand

- **Schritt 2 – Übergabe-Checkliste (vom Kunden bestätigt)**
  - Automatisch passend zur Reparatur – z.B.:
  - **Nach Reifenservice**: Einlagerungsschein, 50-km-Nachziehen erklärt
  - **Nach Bremsenarbeit**: Einfahr-Hinweis erhalten, Probefahrt OK
  - **Nach HU/AU**: TÜV-Bescheinigung + neue Plakette
  - **Nach Inspektion**: Scheckheft aktualisiert, Service-Intervall zurückgesetzt
  - **Immer**: Fahrzeug sauber, Arbeiten erklärt, Rechnung erhalten, Sichtkontrolle OK

- **Schritt 3 – Zufriedenheit + Beanstandungen**
  - ★★★★★ Kundenbewertung direkt bei Übergabe
  - Freitext: Lob / Anmerkungen
  - Beanstandungen → Status automatisch „Unter Vorbehalt"

- **Schritt 4 – Unterschriften**
  - Kunde signiert auf dem Tablet (Pflicht)
  - Werkstatt gegenzeichnet
  - Rechtstext zur Fahrzeugentgegennahme

**Vorteile:**

| Problem | Beweis im Protokoll |
|---------|---------------------|
| „Ich hatte 2 Schlüssel abgegeben" | Kunde hat 2 bestätigt + unterschrieben |
| „Die Reparatur war doch nicht OK" | Sichtkontrolle + Signatur vor Ort |
| „Der TÜV-Bericht fehlt" | Checkpunkt „TÜV-Bescheinigung erhalten" abgehakt |
| „Im Auto war mein Handy" | „Zubehör/Gegenstände zurück" ausdrücklich bestätigt |
| „Die Bremsen machen wieder Geräusche" | „Einfahr-Hinweis erhalten" dokumentiert |

**Ergebnis:** drastisch weniger Reklamationen, rechtssichere Dokumentation, messbare Kundenzufriedenheit.

→ **Von der Anfahrt bis zur Abholung lückenlos digital.**

# Slide 31 – Installation

## In 10 Minuten einsatzbereit

```bash
git clone <repo> werkstatt-termin
cd werkstatt-termin
npm install && npm install --prefix backend && npm install --prefix frontend
```

Dann `backend/.env` konfigurieren (Werkstatt-Name, E-Mail-Server, …) und starten:

```bash
cd backend && node src/index.js      # Port 4100
cd frontend && npm run dev           # Port 5173
```

Login: `admin@werkstatt.local` / `admin123`

**Produktions-Deployment**:
- 1 Server (VPS, 4 GB RAM reicht)
- Nginx/Caddy als Reverse Proxy mit SSL

## Tipps für die Präsentation

**Dauer:** 15–22 Minuten für das volle Deck, 5–7 Minuten für die Kurzform (Slides 1, 3, 6, 11, 18, 20, 24–25 bei Reifen-Fokus, 32, 37).

**Format-Tipps:**

- Screenshots vor der Präsentation aktuell erzeugen (Dashboard mit schönen Daten → Demo-Daten laden!)
- Lass bei Slide 6–8 ein echtes Dashboard live mitlaufen, nicht nur Screenshots – das wirkt beeindruckender
- Bei Slide 11 (KI-Scan) live einen Fahrzeugschein hochladen – das ist der "Wow"-Moment
- Slide 20 (Online-Buchung): Widget live auf einer Testseite zeigen, und während du redest, macht jemand aus dem Publikum die Buchung mit dem Handy
- Slide 32 (ROI): unbedingt individuell durchrechnen für den konkreten Kunden, wenn möglich
- Slides 24–25 (Reifen / Lager): für Reifen- oder Karosserie-Schwerpunkte einplanen; Saison-Mails nur mit echtem SMTP sinnvoll vorführen

**Was du immer dabei haben solltest:**
- Laptop mit Demo-Daten geladen
- Mobiles Hotspot (Internet!) für KI-Features
- Ein echter Fahrzeugschein zum Live-Scannen
- Fallback: Video-Aufnahme der Demo, falls Live-Demo ausfällt
